# 黑客松游戏方案：AI 变异进化肉鸽

## 1. 核心想法

我们希望开发一款网页端轻量肉鸽游戏，主题是“通过食物、药物和未知实验不断进化出一个独一无二的生命体”。玩家从一个非常简单的火柴人或原始小人开始，在一轮轮战斗、选择和变异中逐步获得器官、肢体、能力与副作用，最终形成一个可爱但有点抽象的变异人、小怪兽，或在保守选择下成长为普通人类形态。

游戏的核心不是传统意义上的严肃进化，而是一种带有荒诞感、黑色幽默和 AI 随机性的“培养实验”。玩家每次通关或失败后，都能看到自己这一局培养出的最终形态：它可能长出额外手臂、昆虫翅膀、发光胃袋、机械义肢、蘑菇头、尾巴、触角，也可能因为选择过于保守而维持接近普通人的样子。

整体风格建议控制在“怪但可爱”，避免过度恶心或猎奇。比如“拉肚子”这种负面状态可以包装成夸张的卡通效果：角色短时间降低移动速度，但会在身后留下带范围伤害的毒液区域或臭气云；药物变异可以表现为基因光效、器官贴图变化、能力图标变化，而不是写实的身体恐怖。

## 2. 游戏循环

单局流程可以设计为：

1. 玩家进入一小关战斗房间。
2. 使用当前角色能力击败敌人，获得经验、金币或突变点。
3. 战斗结束后进入“进化选择”界面。
4. 系统随机给出 3 个选项，例如食物、药物、实验试剂。
5. 玩家选择其中一个，获得一个增益和一个可能的副作用。
6. 角色外观、属性、技能或攻击方式发生变化。
7. 重复若干轮后挑战最终 Boss。
8. 结算展示玩家培养出的最终变异体、关键选择路线和能力组合。

这个循环适合黑客松，因为它可以先做得非常小：只要有移动、攻击、敌人、进化三选一、外观拼装、结算展示，就已经能表达完整概念。

## 3. 进化系统设计

进化选择建议分成三类：

### 食物

食物偏向自然成长和身体变化，通常比较稳定，但副作用偏生活化或体质变化。

示例：

- 辣味火锅：攻击附带火焰伤害；副作用是周期性喷火失控，可能短暂暴露位置。
- 变质牛奶：获得毒素抗性；副作用是移动时留下臭气云，敌人受伤但自己速度下降。
- 深海鱼子：长出鱼鳍，提高闪避；副作用是离开水形态后攻击间隔变长。
- 巨型胡萝卜：视野变大，暴击率提升；副作用是更容易被远程敌人发现。

### 药物

药物偏向强力、不可控和基因突变，适合做高风险高收益。

示例：

- 肌肉针剂：攻击力大幅提升；副作用是体型变大，更容易被命中。
- 神经兴奋剂：移动速度和攻击速度提升；副作用是控制会偶尔反向或短暂眩晕。
- 未知基因药：随机长出一个器官或肢体；副作用随机。
- 再生药膏：每关回血；副作用是部分攻击力下降。

### 实验物

实验物可以承担最抽象、最 AI 的部分，让每局出现更不可预期的组合。

示例：

- AI 梦境胶囊：由 AI 生成一条奇怪的突变描述，并映射到实际 buff。
- 混沌培养液：随机合成两个已有能力，但生成一个新的弱点。
- 记忆罐头：复制上一次选择的能力，但副作用翻倍。
- 迷你黑洞糖：攻击吸附敌人；副作用是偶尔把掉落物也吸走。

## 4. AI 结合点

为了更贴合腾讯黑客松主题，AI 不应该只作为“生成图片的工具”，最好也进入玩法表达。

建议优先做以下几个 AI 结合点：

1. AI 生成变异描述：每次进化后生成一句有趣的文字，例如“你的胃开始像小型反应堆一样发光”。
2. AI 生成外观关键词：根据玩家选择的食物、药物和已有器官，生成角色外观标签，例如“发光胃袋、短角、软糖质感、三只小手”。
3. AI 辅助生成结算档案：通关后生成一份“实验报告”，描述这个生命体的特征、性格、危险等级和培养路线。
4. AI 资产生产：提前用 AI 生成贴图、角色部件、敌人、Boss、背景和 UI 图标，再放入游戏中组合使用。

黑客松 MVP 阶段不建议在战斗中实时调用大模型，因为网络、延迟、成本和稳定性都会增加风险。更稳妥的方案是：游戏逻辑本地可运行，AI 只负责“选择后的文案生成”和“结算报告生成”；如果接口不可用，则使用本地预设模板兜底。

## 5. 最小最快实现方案

### MVP 目标

做一个 5 到 10 分钟可以完整体验的网页端游戏原型。

最小功能：

- 一个可移动角色。
- 自动攻击或鼠标点击攻击。
- 简单敌人会向玩家移动。
- 击败敌人获得经验。
- 每升一级弹出 3 个进化选项。
- 每个选项包含一个 buff 和一个 debuff。
- 角色外观通过贴图部件叠加变化。
- 第 5 或第 8 波出现 Boss。
- 击败 Boss 后展示最终变异体和 AI 生成的实验报告。

### 最小内容量

角色：

- 初始火柴人或普通小人 1 个。
- 可叠加身体部件 8 到 12 个，例如角、翅膀、尾巴、额外手臂、发光胃、机械腿、蘑菇帽、眼球、护甲、触角。

敌人：

- 普通近战敌人 1 种。
- 快速小怪 1 种。
- 远程敌人 1 种。
- Boss 1 个。

进化选项：

- 第一版准备 20 到 30 个固定选项即可。
- 每个选项写清楚 `name`、`type`、`buff`、`debuff`、`visualParts`、`descriptionPrompt`。

地图：

- 一个固定竞技场即可。
- 可以用简单背景图、障碍物和边界制造空间感。

## 6. 推荐技术栈

### 前端游戏

推荐使用：

- TypeScript
- Vite
- Phaser 3

原因：

- Phaser 3 是成熟的 HTML5 2D 游戏框架，适合网页端、肉鸽、小型动作游戏。
- 比 Unity 更轻，不需要复杂打包和 WebGL 体积负担。
- 资源管理、碰撞、动画、输入、场景切换都比较直接。
- 黑客松阶段更容易快速出结果。

如果团队更熟 Python，可以让 Python 主要负责后端和 AI 接口，不建议用 Python 写浏览器端游戏主体。

### 后端

推荐使用：

- Python
- FastAPI

后端职责：

- 调用 AI 接口生成进化描述、怪物档案、结算报告。
- 保存玩家本局变异路线。
- 提供简单排行榜或分享码。
- 提供 AI 失败时的本地模板兜底。


## 7. 视觉资产建议

视觉设计建议采用“AI 生成静态贴图 + 程序拼装”的方式，而不是一开始追求完整动画。

推荐资产拆分：

- 身体底图：火柴人、小人、基础怪兽身体。
- 部件贴图：头、眼睛、角、手臂、腿、尾巴、翅膀、器官、背包、药剂管。
- 特效贴图：火焰、毒雾、臭气云、治疗光、基因光圈、爆炸。
- UI 图标：食物、药物、实验物、buff、debuff。
- 背景图：实验室、培养皿、废弃厨房、基因工厂。

角色外观不要每次整张重画，而是把 AI 生成的部件做成透明背景 PNG，再在游戏中按层级叠加。例如：

1. 底层：身体。
2. 中层：手脚、尾巴、翅膀。
3. 上层：头部器官、眼睛、角、发光部件。
4. 特效层：毒雾、火焰、光环。

这样可以用很少的图片组合出大量变化，也更符合“每局培养出不同怪物”的主题。

AI 生成提示词可以统一成类似格式：

```text
cute weird mutant creature body part, transparent background, 2D game asset, soft cartoon style, clean outline, colorful, front view, no text
```

如果要生成偏恶搞的能力图标，可以用：

```text
cartoon game icon of a strange food mutation, cute gross humor, colorful, clean vector-like style, no text, square icon
```

注意事项：

- 尽量要求透明背景，方便贴到游戏里。
- 保持统一视角，比如全部正面或 3/4 视角。
- 保持统一描边和色彩风格。
- 不要生成过度写实、血腥或恐怖的素材，容易影响观感和审核风险。
- 每个部件图片尺寸保持一致，例如 512x512，再在程序里缩放。

## 8. 数据结构建议

进化选项可以先用 JSON 配置驱动，方便快速扩展。

```json
{
  "id": "spoiled_milk",
  "name": "变质牛奶",
  "type": "food",
  "buff": {
    "poisonDamage": 8,
    "poisonRadius": 80
  },
  "debuff": {
    "moveSpeedMultiplier": 0.9
  },
  "visualParts": ["glowing_belly", "green_gas"],
  "aiPromptTags": ["变质牛奶", "毒雾", "肚子咕噜叫", "可爱怪兽"]
}
```

玩家本局状态可以记录：

```json
{
  "runId": "run_001",
  "level": 5,
  "choices": ["spoiled_milk", "muscle_injection", "tiny_black_hole_candy"],
  "stats": {
    "hp": 120,
    "damage": 18,
    "speed": 110
  },
  "visualParts": ["glowing_belly", "extra_arm", "black_hole_aura"]
}
```

这样结算页可以把选择路线发给后端，由 AI 生成“实验报告”。

## 9. 初期开发排期

### 第一天

- 搭建 Vite + Phaser 项目。
- 实现玩家移动、敌人追踪、碰撞和生命值。
- 实现基础攻击和经验升级。
- 做出进化三选一 UI。

### 第二天

- 加入 20 个进化配置。
- 实现 buff/debuff 系统。
- 实现角色贴图部件叠加。
- 加入 Boss 和胜利结算。

### 第三天

- 接入 Python FastAPI 后端。
- 接入 AI 文案生成或准备 mock 接口。
- 完成实验报告结算页。
- 替换 AI 生成的美术素材。
- 调整手感、数值和展示效果。

如果只有更短时间，可以先砍掉后端，使用前端本地模板生成文案，确保游戏闭环完整。

## 10. 风险与取舍

主要风险：

- AI 实时生成图片不稳定，耗时长，不适合作为游戏运行时核心功能。
- 进化组合太多会导致数值失控。
- 视觉风格如果太恶心，可能影响展示效果。
- Web 端动作游戏如果手感不好，玩家会很快失去兴趣。

建议取舍：

- 先保证 5 分钟完整闭环。
- 先做固定配置，再加 AI 随机描述。
- 先用部件拼装表达变异，不做复杂骨骼动画。
- 恶心元素保持卡通化，重点放在“抽象、荒诞、可爱、好笑”。
- AI 要服务于“每局都是独一无二的培养结果”，而不是为了接 AI 而接 AI。

## 11. 一句话提案

这是一款 AI 驱动的网页端变异进化肉鸽游戏：玩家通过食物、药物和实验物不断选择高风险进化，在战斗中获得能力和副作用，最终培养出一个独一无二、怪得可爱的小生命体，并由 AI 生成它的专属实验报告。
