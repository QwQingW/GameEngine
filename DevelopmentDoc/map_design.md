# 🧬 变异吧！人类！—— 地图场景设计文档

## 整体画风定位

- **风格**：像素风 (Pixel Art)，偏卡通向、抽象化
- **色板策略**：每张地图使用独立的调色板，随着关卡推进色彩从冷→暖→污染→霓虹→冷峻，体现环境演变
- **画面比例**：俯视/顶视角 (Top-down view)，适配当前游戏引擎的 800×600 玩法区域
- **动态要素**：背景层静态铺设 + 前景层可交互障碍物/资源 + 环境粒子特效

## 关卡地图设计

### 第一关：原始海洋 (Primordial Ocean)

#### 主题概念
生命起源于海洋。这是一个深海热液喷口区域的场景，充满了有机汤的混沌与原初生命的力量。蓝色与深绿色为主调，发光浮游生物点缀其间。

#### 场景描述

**背景层（静态）**：
- 深蓝到墨绿的渐变水体，模拟深海无光环境
- 远处散布发光的**热液喷口**，喷出深色矿物烟雾，带有微弱的橙红色光晕
- 海底地面为不规则的火山岩纹理，混杂玄武岩柱状节理
- 星星点点的**生物荧光粒子**——绿色、蓝色的浮游微生物在水中缓慢飘浮
- 远处有成片的**叠层石**（蓝藻形成的柱状结构），暗示早期光合作用的出现

**障碍物（可碰撞）**：
- **玄武岩石柱**（1×1格）- 海底火山喷发形成的六角形石柱，不可破坏
- **珊瑚礁群**（2×2格）- 原初的管状珊瑚，碰撞后短暂减速玩家
- **热液喷口**（1×1格）- 间歇喷发高温矿物蒸汽，站在附近持续扣血
- **水母群**（1×1格）- 半透明发光水母上下漂浮，触碰麻痹1秒

**资源点（可拾取/互动）**：
- **有机汤球**——发光的绿色球体，拾取+20HP（原始生命物质的回血效果）
- **矿物结晶**——发光的紫色晶簇，拾取后下次攻击附带额外5点伤害
- **氧气气泡**——从海底裂缝中升起的大气泡，拾取后移动速度短暂+20%

**生物/氛围细节**：
- 屏幕边缘偶尔游过**原始鱼类**（外形类似皮克鱼或奇虾）的剪影
- 水体有轻微的波动扭曲效果（像素级的水纹着色器）
- 气泡不断从地面裂缝和热液口向上飘升

### 第二关：变异丛林 (Mutant Jungle)

#### 主题概念
生命登陆后的爆炸式辐射进化。一片被基因实验废料污染的疯狂丛林，植物巨大化、荧光化，生物形态扭曲怪异。绿色与紫色为主调，荧光色做点缀。

#### 场景描述

**背景层（静态）**：
- 遮天蔽日的**巨型蕨类植物**和奇异真菌，形成幽暗的林下环境
- 某些植物叶片呈现不自然的**荧光紫/荧光粉红色**，暗示基因突变
- 地面覆盖厚厚一层腐殖质，夹杂发光的菌丝网络（类似《阿凡达》潘多拉星球的荧光地面）
- 粗壮的藤蔓缠绕着古木树干，藤蔓上有吸盘状结构
- 间歇有**孢子云**在林间飘散——黄绿色的小颗粒聚合体

**障碍物（可碰撞）**：
- **食人花**（2×2格）- 张合的大嘴花，周期性咬合，站在前方扣血
- **荆棘藤蔓**（1×N横向/纵向）- 带刺的粗藤，不可穿越，碰撞扣少量血
- **毒蘑菇群**（2×2格）- 紫色伞盖的巨大蘑菇，触碰后释放毒雾（持续5秒减速区域）
- **变异蚁穴**（1×1格）- 土堆状结构，周期性喷出小酸液球，碰到扣血

**资源点（可拾取/互动）**：
- **荧光浆果**——挂在灌木上的发光橙色果实，拾取+25HP
- **突变孢子**——浮在空中的紫色发光孢子，拾取后暴击率临时+15%
- **远古树脂**——树干上渗出的金色胶状物，拾取后攻击速度临时+30%
- **DNA藤蔓**——带有螺旋纹路的蓝色藤蔓，拾取获得额外EXP

**生物/氛围细节**：
- 巨大的**变异昆虫**剪影在背景树冠层飞过（如发光的巨型蜻蜓）
- 地面菌丝有**呼吸般的光效**——明暗交替闪烁
- 雨林特有的间歇**像素雨滴**从树叶间滴落
- 偶尔出现的**变异青蛙**——三条腿、荧光皮肤的像素小动物在场景中跳跃

### 第三关：基因污染城市 (Contaminated City)

#### 主题概念
基因实验失控后，城市开始"生物化"。混凝土裂缝中钻出藤蔓，路灯杆被生物膜覆盖，废弃的汽车上长满了菌菇。灰色为主调，橙红污染和绿色生物质做点缀。

#### 场景描述

**背景层（静态）**：
- 灰暗的**城市天际线**——密集的居民楼和工厂烟囱，天空呈黄褐色雾霾色调
- 建筑外墙上爬满了**紫红色生物组织**，窗户中透出诡异的绿光
- **废弃的化工厂**烟囱还在冒烟（像素黑烟动画），烟柱有微微的绿色
- 街道上散落着废弃车辆、翻倒的垃圾桶、碎裂的路面
- **基因污染的警示标志**贴在各处——黄色三角形带生物危害符号
- 路灯忽明忽暗地闪烁（像素灯光闪烁效果）

**障碍物（可碰撞）**：
- **废弃汽车**（3×1格）- 生锈的车辆残骸，不可穿越，可以推着移动
- **化学废料桶**（1×1格）- 辐射标志的油桶，碰撞后爆炸（范围伤害20，油桶消失）
- **生物质路障**（2×2格）- 警察路障被粉色生物膜覆盖，会缓慢再生HP（模拟活体组织）
- **崩塌的建筑物残骸**（3×3格）- 大块混凝土碎块，完全阻挡通行
- **漏电的电线杆**（1×1格）- 歪斜的电线杆，周围有电火花，站在附近周期性被电击

**资源点（可拾取/互动）**：
- **急救包**——医疗箱造型的红色+号图标，拾取+30HP
- **废弃实验药剂**——装在有荧光绿色液体的试管，拾取后攻击力+8持续15秒
- **电池核心**——从废弃机器中掉落的蓝色能量块，拾取后技能CD减少
- **抗污染面具**——白色防毒面具图标，拾取后免疫毒雾/污染区域15秒

**生物/氛围细节**：
- 下水道井盖冒出**绿色毒气**（像素烟雾粒子上飘）
- 街道角落有**变异的城市动物**——长着多只眼睛的老鼠、发光的蟑螂群
- 建筑表面时不时有**生物脉动**——粉色生物膜的扩张收缩动画
- 背景远处有直升机飞过，探照灯扫过街道
- 偶尔发生的**基因风暴**——屏幕短暂变为淡紫色，玩家视野范围略微缩小

### 第四关：赛博进化城 (Cyber-Evolution Nexus)

#### 主题概念
未来的极端——科技与基因融合的超级都市。纳米机器与有机体共存的赛博朋克世界。深紫与霓虹蓝为主调，RGB色光污染做点缀。

#### 场景描述

**背景层（静态）**：
- 高耸入云的**摩天大楼**，表面被全息投影和霓虹广告覆盖（像素滚动或闪烁的霓虹招牌）
- 巨型**全息屏幕**播放"基因升级"广告——"Want more arms? Upgrade today!"
- 天空中飘着**纳米云**——半透明的霓虹蓝色微粒群缓缓流动
- **飞行汽车**在高楼间穿梭，留下粉色/蓝色的像素光轨
- 街道地面是发光的**网格地板**（类似Tron风格），数据流在格线中流动
- 建筑角落堆满了**电子废弃物**和闪烁的服务器机柜

**障碍物（可碰撞）**：
- **自动防御炮塔**（2×2格）- 检测到玩家后周期性发射能量弹，可被摧毁（HP=50）
- **纳米屏障墙**（1×3格）- 半透明的蓝色能量墙，周期性开关（2秒开2秒关），关闭时可通过
- **数据防火墙**（1×1格）- 橙红色的全息六边形护盾，不可穿越，可被攻击破坏
- **废弃机甲残骸**（3×2格）- 破损的军用机甲，部分遮挡通行
- **基因扫描门**（2×1格）- 类似安检门，通过时触发扫描，若玩家有3个以上进化则触发警报（召唤一波敌人）

**资源点（可拾取/互动）**：
- **纳米修复群**——发光的蓝色纳米微粒聚集球，拾取后每秒回血+3HP持续8秒
- **数据水晶**——紫红色的六面体水晶，拾取+50EXP
- **过载核心**——从服务器中拆出的发热能量体，拾取后攻击力翻倍但每秒扣3HP，持续10秒
- **基因黑客芯片**——绿色电路板图标，拾取后下一次进化可以四选一而非三选一

**生物/氛围细节**：
- 下水道口冒出**蒸汽**，霓虹灯光的反射让蒸汽呈现多彩效果
- 到处可见**赛博改造人NPC**——半机械半生物体在背景中蹒跚行走
- 全息广告牌偶尔**闪烁故障**——画面撕裂、色彩偏移的glitch效果
- 地面数据网格线有**脉冲波**周期性从城市中心向外扩散
- 空中飘浮着**纳米尘埃**——微小的发光颗粒慢速漂浮
- 背景音视觉化：霓虹招牌的电流声、全息广告的电子音、远处的警笛声

### 第五关：生化实验室 (Biohazard Lab - "Genesis Chamber")

#### 主题概念
一切的源头——基因工程的终极实验室。类似《生化危机》的蜂巢研究所。白色无菌环境被彻底破坏，血迹斑斑，管道破裂，实验体逃逸。惨白冷光为主调，血红和警报红灯做点缀。这是游戏的最终关卡。

#### 场景描述

**背景层（静态）**：
- 惨白的**无菌走廊**——白色/灰色的医院风墙壁，地面为黑白格瓷砖（类似《闪灵》走廊）
- 墙壁上到处是**血色手印**和**抓痕**（像素化的血迹纹理）
- 天花板上挂着**破裂的培养罐**——透明玻璃罐，部分还在滴落绿色培养液
- 应急**红色警报灯**旋转闪烁（像素点阵式的红蓝交替光效），在走廊中投下移动的红色光斑
- 墙上的**安全警告标志**和**基因图谱海报**被撕烂或被血迹覆盖
- 走廊两侧有**巨大的金属门**，上面写着 "BIOHAZARD LEVEL 4"、"GENESIS PROJECT"、"仅限授权人员"
- **监控摄像头**（已破损）挂在角落，镜头碎裂但仍在转动
- 远处可见**大型基因测序仪**的轮廓，屏幕显示ERROR

**障碍物（可碰撞）**：
- **倾倒的培养罐**（2×2格）- 半碎的玻璃圆筒，溢出绿色培养液（站在液体中持续扣血）
- **防爆门**（3×1格）- 厚重的金属安全门，需要击破（HP=80）才能通过
- **破试管堆**（1×1格）- 地面上的碎玻璃，走过时扣5HP
- **故障机械臂**（2×2格）- 从天花板上垂下的实验机械臂，随机摆动，被击中扣血
- **激光栅栏**（1×N格）- 红色激光束组成的栅栏，周期性开关（1.5秒开1.5秒关），类似电影中的安保激光

**资源点（可拾取/互动）**：
- **基因血清注射器**——带有蓝色液体的注射器，拾取后HP全满且体型+10%
- **反突变抑制剂**——绿色药丸瓶，拾取后清除所有负面Debuff
- **原始基因样本**——发光的金色双螺旋DNA瓶，拾取后永久+3攻击力（本关最强资源）
- **实验日志芯片**——红色数据卡，拾取+100EXP，附带一段剧情文本

**BOSS房 - 基因起源室 (Genesis Chamber)**：
- 一个巨大的圆形房间，中央是**"生命之树"基因装置**——精密而庞大的基因合成仪器
- 房间周围排列着**培养罐阵列**，每个罐子里都有不同进化阶段的生物标本
- 天花板是一整面**基因图谱投影**，扭曲变异的DNA双螺旋在旋转
- 地板是发光的**基因序列网格**，流动的ATCG碱基对代码（像素滚动效果）
- BOSS出场时培养罐全部破裂，液体喷涌而出
- 房间四角各有**应急喷淋装置**，间歇喷出冷却蒸汽遮蔽视线

**生物/氛围细节**：
- 破裂的管道喷出**白色蒸汽**（像素粒子），在警报红光中呈血色
- 培养罐中偶尔有**不明生物**游过——触手拍打玻璃
- 墙上显示屏不断闪烁**ERROR: CONTAINMENT BREACH**、**SUBJECT ESCAPED**等警告
- 血迹从某个房间门缝下渗出，缓慢扩散
- 广播系统播放着循环的机械女声："Warning. Bio-hazard containment breach. All personnel evacuate immediately."
- 房间角落有**倒地的研究员NPC**（像素尸体），身上有实验服和胸牌
- 最终BOSS出现前，基因起源室的灯光全部熄灭3秒，然后所有培养罐同时爆裂

## AI工具推荐 —— 像素风动态场景生成

### 核心思路：AI生成素材 + Phaser引擎渲染动态

由于项目使用 **Phaser 3** 游戏引擎，最务实的方案是：
- **静态素材**（背景图、tileset、障碍物精灵图）→ 用AI生成
- **动态效果**（粒子、光照闪烁、水流、全息特效）→ 在Phaser中用代码实现
- **可选：GIF/视频背景** → 用AI视频工具生成后作为纹理播放

---

### 〇、静态资源文件格式规范

#### 结论：PNG-24 是最佳选择

| 格式 | 推荐度 | 透明度 | 压缩 | 像素风适用性 | 说明 |
|------|--------|--------|------|-------------|------|
| **PNG-24** | ⭐⭐⭐⭐⭐ | ✅ 全通道 | 无损 | 完美 | 像素艺术的事实标准，Phaser 原生优先支持 |
| **PNG-8** | ⭐⭐⭐⭐ | ✅ 布尔透明 | 无损 | 很好 | 256色以内的小型精灵适用，文件更小但颜色受限 |
| **WebP** | ⭐⭐⭐ | ✅ 全通道 | 有损/无损 | 一般 | 现代浏览器均支持，但有损压缩会模糊像素边缘 |
| **JPEG** | ❌ | ❌ 不支持 | 有损 | 差 | 无透明通道，压缩伪影破坏像素锐利度，不推荐 |
| **SVG** | ❌ | ✅ | N/A | 不适用 | 矢量和像素风天然冲突，Phaser 不支持直接渲染 |
| **GIF** | ⭐⭐ | ✅ 布尔透明 | 无损 | 可用 | 仅256色，只适合小型粒子精灵或简易动画帧 |

#### 为什么 PNG 是像素游戏的不二之选

1. **无损压缩**：每个像素颜色精确保持，不会像 JPEG 那样出现模糊伪影
2. **Alpha 透明通道**：支持 0-255 级半透明，粒子、光晕、全息特效都依赖这个
3. **Phaser 原生支持**：`this.load.image()` 默认处理 PNG，零配置
4. **Aseprite / Pixelorama** 等行业工具的标准导出格式

#### WebP 的权衡

WebP 作为现代替代方案，在文件大小上有优势（通常比PNG小25-35%），但：
- 有损 WebP 会轻微模糊像素边缘（像素风的大忌）
- 需要无损模式下才适合像素艺术，但无损 WebP 文件优势不明显
- Phaser 支持，但需要确保 Vite 构建配置正确

> **本项目建议**：开发阶段全部用 PNG，如果最终打包体积过大，再考虑将背景大图转为无损 WebP。

---

#### 精灵图组织方式：Texture Atlas（推荐）

不要把每个精灵存成独立文件，而是打成**纹理图集 (Texture Atlas)**：

```
方式A：Sprite Sheet（均匀网格精灵表）
┌────┬────┬────┬────┐
│ 🌊  │ 🫧  │ 🦪  │ 💎  │    4×4 网格，每个 cell 32×32
├────┼────┼────┼────┤    总图：128×128 PNG
│    │    │    │    │
└────┴────┴────┴────┘

方式B：Texture Atlas（不规则打包）
  紧凑排列，附带 JSON 描述每个精灵的坐标和尺寸
  更省空间，但需要工具生成
```

| 方式 | 文件 | 适用场景 | Phaser 加载 |
|------|------|----------|------------|
| 独立文件 | `coral.png`, `rock.png`, ... | 原型/Demo 阶段，快速迭代 | `this.load.image()` |
| Sprite Sheet | `ocean_sprites.png` | 简单动画帧序列，统一尺寸 | `this.load.spritesheet()` |
| Texture Atlas | `ocean_atlas.png` + `ocean_atlas.json` | 发布/优化阶段，不规则尺寸 | `this.load.atlas()` |
| 多页图集 | `ocean_0.png` + `ocean_0.json`, ... | 资产量大时自动分页 | `this.load.multiatlas()` |

**推荐节奏**：前期用独立文件快速验证 → 正式版用 TexturePacker 打包为 atlas。

#### 像素风的尺寸与缩放规范

```
像素颗粒基准：每个关卡统一一个基础像素单位
├── 粒子/小型特效    8×8 或 16×16
├── 小型资源/道具    16×16 或 24×24
├── 中型障碍物       32×32 或 48×48
├── 大型障碍物       64×64 或 96×96
├── Tile 单元        32×32（标准）
├── Tileset 整图     512×512 或 1024×1024（约16×16或32×32个tile）
└── 完整场景背景     1600×1200（按地图物理尺寸等比放大）

关键规则：所有尺寸必须是2的幂次（32, 64, 128, 256, 512, 1024）
          → GPU 纹理处理性能最优
          → 整数倍缩放不会产生模糊
```

#### 防止像素被"平滑"的三道防线

像素风最怕的就是图片被缩放时变模糊，需要三层保障：

```typescript
// 1. Phaser 加载时：关闭抗锯齿
this.load.image('tileset');
this.textures.get('tileset').setFilter(Phaser.Textures.FilterMode.NEAREST);

// 2. Phaser 游戏配置中：全局设置
const config: Phaser.Types.Core.GameConfig = {
    render: {
        pixelArt: true,           // ← 这一行会全局禁用平滑
        antialias: false,
        roundPixels: true,        // ← 避免像素对齐导致抖动
    }
};

// 3. CSS 层面（如果使用了DOM元素）：
// canvas { image-rendering: pixelated; }
```

### 二、动态/动画场景生成

动态场景的实现分为三个层级，按成本从低到高排列：

#### 层级1：Phaser引擎原生动画（推荐 ⭐⭐⭐⭐⭐）

**完全不依赖AI**，在引擎中实现，性能最好，最适合游戏：

- **粒子系统** (`Phaser.GameObjects.Particles`)：
  - 海洋气泡、丛林孢子、城市毒气、赛博纳米尘、实验室蒸汽
  - 每个场景只需AI生成1-2帧粒子精灵图即可
  
- **Tween动画** (`Phaser.Tweens`)：
  - 霓虹灯闪烁、全息广告滚动、警报灯旋转、水纹波动
  - 热液喷口呼吸光效、菌丝脉动、激光栅栏开合

- **Tilemap动画** (`Phaser.Tilemaps`)：
  - 使用spritesheet导入，定义动画帧序列
  - 适合水面波纹、岩浆流动、数据网格脉冲

- **Shader/滤镜** (`Phaser.Renderer.WebGL.Pipelines`)：
  - 水下扭曲效果、赛博glitch效果、热浪扭曲

> **结论**：90%的动态需求都可以在Phaser中实现，只需要AI生成静态素材即可。

#### 层级2：AI生成动画素材（推荐 ⭐⭐⭐⭐）

适合需要复杂动画但不想手绘的场景：

| 工具 | 适合场景 | 输出格式 | 说明 |
|------|----------|----------|------|
| **Pika Labs** | 场景氛围动画 | MP4/GIF | `pixel art underwater scene with bubbles rising` → 视频纹理 |
| **Runway Gen-3** | 高质量场景动画 | MP4 | 可精确控制运动，生成1280×720以上分辨率 |
| **Luma Dream Machine** | 3D到视频的场景 | MP4 | 从概念图生成带景深的动画，沉浸感强 |
| **Kling (可灵)** | 国内最优视频生成 | MP4 | 国产工具，中文提示词支持好，生成速度快 |
| **海螺AI** | 国内视频生成 | MP4 | MiniMax出品，动态场景质量高 |

**使用方式**：
1. 先用Midjourney/Leonardo生成静态场景图
2. 用上述工具将静态图转为动态视频
3. 在Phaser中通过 `Phaser.Video` 对象播放作为背景层

> ⚠️ 注意：视频背景的文件大小较大（单个场景可能10-50MB），需要预加载管理。建议只在关键场景（如BOSS房）使用。

#### 层级3：AI实时生成（探索性 ⭐⭐⭐）

| 工具 | 说明 |
|------|------|
| **ComfyUI + AnimateDiff** | 本地搭建流程，可实现条件化实时生成，但延迟较高(2-5秒/帧) |
| **Gen-3 Turbo API** | Runway的实时API，延迟1-3秒/帧，适合非实时预渲染回放 |

> ⚠️ 目前实时AI生成的延迟还无法满足60fps游戏需求，仅适合预渲染场景。
